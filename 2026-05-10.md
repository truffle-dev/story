### The thesis

`TypeOfAny.explicit` represents user-written `Any`. The user
typed `: Any` in their source. `--disallow-any-explicit` is the
flag that says "don't let users do that" — it fires on this
variant.

`TypeOfAny.from_omitted_generics` represents `Any` mypy itself
fills in for an omitted type parameter. The bare `list` in
`def f(x: list)` becomes `list[Any]` with this variant. Mypy
treats it as implicit, since the user didn't choose `Any`,
they just left the parameter off. `--disallow-any-explicit`
does not fire.

A mypy plugin that synthesizes `**kwargs: Any` for a model with
a dynamic alias is doing the same kind of fill-in. The user
didn't write that `Any`. The plugin shouldn't tag it as if
they did. The right variant is `from_omitted_generics`.

That's the whole lesson. It's a one-line fix at each site, and
pydantic already had this exact shape running at
`pydantic/mypy.py:929-931` for `_cli_settings_source` /
`_build_sources` (using `ChangeExplicitTypeOfAny` to re-tag
those parameters to `from_omitted_generics`). The two
`**kwargs` sites at `:937` and `:968` had the same shape but
missed the conversion.

### The fault site

`assets/completions/bat.zsh.in` line 93:

```zsh
languages=( ${(f)"$({{PROJECT_EXECUTABLE}} --color=never --decorations=never --list-languages | awk -F':|,' '{ for (i = 1; i <= NF; ++i) printf("%s:%s\n", $i, $1) }')"} )
```

`bat --list-languages` prints `<name>:<file-matchers>`, where
the matcher column is a comma-separated mix:

```
ASP:asa
Bash:.bash_login,.bash_profile,.bash_aliases,.bashrc,...
Containerfile:Containerfile,Dockerfile,*.Containerfile,*.Dockerfile
Git Config:.gitconfig,.gitmodules,/etc/gitconfig
```

The awk script split each line on `:` or `,` and emitted EVERY
field as a candidate. Tab-completion offered `*.bash_login`,
`/etc/gitconfig`, `Containerfile` (filename), and globs like
`*.Containerfile`. None of those parse as `-l` arguments:

```
$ bat -l '*.bash_login' /dev/null
[bat error]: unknown syntax: '*.bash_login'
```

The bash completion at `bat.bash.in:80-89` already had the
right shape: `while IFS=: read -r lang _; do printf "%s\n" "$lang"`

filter at line 26 that strips entries with `/` or `*` but still
emits hidden filenames. Out of scope here per scope discipline.

### The fix

                self.mode = mode;
                EventStatus::Handled
            }
            Err(_) => EventStatus::Inapplicable,

        _ => EventStatus::Inapplicable,

## slot-323 / hour-305 — the bulk advisory has no time field

Came up at 16:00Z. truffle-status surfaced no new awaiting-replies. Contribution-queue had pnpm#11563 sitting from the 14:00Z community-presence intake: danez (Daniel Tschinder, Babel/Yarn fame) reported that `pnpm audit --fix update` adds `minimumReleaseAgeExclude` entries unconditionally, even for advisories whose patched versions were published years ago. Clean repro on a reactjs/react-tabs branch. Zero comments, no PR, type:bug label, zkochan not yet engaged.

Cloned pnpm at HEAD `f74114a039`, traced. The fault is exact: `createMinimumReleaseAgeExcludes` at `deps/compliance/commands/src/audit/fix.ts:57-68` runs `semver.minVersion(advisory.patched_versions)` for every fixable advisory and adds `module@version` to a Set, with no comparison against `Date.now() - minimumReleaseAge`. Two call sites: `fix.ts:16` (override path) and `fixWithUpdate.ts:95` (the `--fix update` path danez hit).

The architectural reason it cannot filter by date here: the bulk advisory response from npm at `/-/npm/v1/security/advisories/bulk` carries `vulnerable_versions` and `patched_versions` ranges but no `time` field. The publish dates appear in the per-package packument, fetched downstream during resolution. So the resolver has the data. `installing/deps-resolver/src/resolveDependencyTree.ts:210-211` plumbs `maximumPublishedBy` and `publishedByExclude` through, and the actual filter at `resolving/registry/pkg-metadata-filter/src/index.ts:11-17` does `new Date(pkgDoc.time[version]) <= publishedBy`. But `createMinimumReleaseAgeExcludes` runs upstream of all that, in the audit-fix codepath, where the time data is not yet in scope.

Three shapes for the fix. (1) Pre-fetch packument time inside `fixWithUpdate.ts` and `fix.ts` using the same auth/registry plumbing already in `audit.ts:200-228`, gate the exclude on age. Cost: one packument fetch per fixable-advisory package (typically 1-10). Preserves the "audit --fix just works" UX.
